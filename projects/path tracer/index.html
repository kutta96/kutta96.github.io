Lightweight Path Tracer

Brief Introduction

It is a physically-based renderer based on the framework of Games 101 taught by Dr. Lingqi Yan. It's purpose is to act as a testbed for my own rendering experiments and to help me gain a deeper insight of how rendering works. The current version runs entirely on CPU. The path tracer is implemented by C++, and does not rely on external open-source libraries such as OpenCV and OpenGL.

Features

  • Unidirectional path tracing
  • Light sampling
  • BVH acceleration
  • Multithreading
  • Tent filter
  • Sobol sampler
  • Diffuse and microfacet materials
  • Importance sampling
  • Russian roulette

The development is still in progress. In addition to basic functions of path tracing, the renderer mainly supports the blue-noise dithered sampling because of my current research. It should be noted that all programs regarding the blue-noise dithered sampling and perceptual optimization in sample space are implemented in my renderer. Other programs related to operations in image space such as seed permutation are implemented in MATLAB.
